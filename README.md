# OBJ 3D Model Viewer

[![CI](https://github.com/Lazarus860451/--OBJ_viewer-based-on-Pyglet-and-ModernGL/actions/workflows/ci.yml/badge.svg)](https://github.com/Lazarus860451/--OBJ_viewer-based-on-Pyglet-and-ModernGL/actions/workflows/ci.yml)
[![GitHub license](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)


OBJ 3D Model Viewer - это легкий, открытый и расширяемый просмотрщик 3D моделей в формате OBJ, написанный на Python. Программа использует ModernGL для графики, Pyglet для создания окна и обработки ввода, а также NumPy для быстрых вычислений. Она может читать OBJ и MTL файлы, отображать 3D модели, управлять камерой и применять текстуры.

# Установка

### Для пользователя

Чтобы запустить программу:

1. Клонировать репозиторий

```bash
git clone https://github.com/Lazarus860451/--OBJ_viewer-based-on-Pyglet-and-ModernGL.git
cd --OBJ_viewer-based-on-Pyglet-and-ModernGL
```

2. Установить зависимости

```bash
pip install -r requirements.txt
```

3. Запустить программу

```bash
python main.py models/cube.obj
```

### Для разработчика

Если Вы хотите внести вклад в проект:

1. Клонировать репозиторий

```bash
pip install -r requirements.txt
pip install flake8 pytest
```

3. Запустить проверку кода

```bash
flake8 src/ tests/ --max-line-length=79
```

4. Запустить тесты

```bash

```

# Описание модулей

### camera.py

Камера реализована в сферических координатах. Параметры камеры:
- `distance` - расстояние до модели
- `rot_x` - угол поворота по вертикали (pitch)
- `rot_y` - угол поворота по горизонтали (yaw)

Методы камеры:
- `get_view_matrix()` - вычисление view матрицы
- `get_projection_matrix()` - вычисление projection матрицы
- `rotate(dx, dy)` - вращение от движения мыши
- `zoom(delta)` - масштабирование от колёсика мыши

### obj_loader.py

Парсер OBJ и MTL файлов. Основные возможности:
- Чтение вершин (`v`), текстурных координат (`vt`), нормалей (`vn`)
- Поддержка разных форматов граней (`f`)
- Триангуляция четырёхугольников
- Чтение материалов из MTL файлов (`Kd`, `map_Kd`)
- Автоматическое вычисление центра и размера модели
- **Оптимизация**: использование NumPy для векторных операций

### renderer.py

Рендеринг с использованием ModernGL:
- Создание VBO (Vertex Buffer Object) и VAO (Vertex Array Object)
- Компиляция шейдеров
- Загрузка текстур
- Отрисовка по группам материалов

### shaders.py

GLSL шейдеры:
- **Вершинный шейдер**: преобразование координат, передача цвета и текстурных координат
- **Фрагментный шейдер**: смешивание текстур с цветами вершин

### timer.py

Утилиты для измерения производительности:
- `start_timer(name)` - начать замер
- `end_timer(name)` - закончить замер
- `print_all_timings()` - вывести все результаты

# Производительность

### Ключевая оптимизация в obj_loader.py

**Проблема**: При загрузке OBJ файла с 140000 вершин время ожидания было большим из-за циклов Python для вычисления цветов вершин.

**Решение**: Замена медленного цикла Python на векторные операции NumPy.

**Результат**: Время обработки сократилось с 7.5 секунд до 1 секунды (в 7.5 раз быстрее).

```python
# До (медленный цикл Python)
colors = []
i = 0
while i < len(vertices):
    y = vertices[i + 1]
    t = (y - min_y) / (max_y - min_y)
    gray = 0.3 + t * 0.6
    colors.append(gray)
    colors.append(gray)
    colors.append(gray)
    i = i + 3

# После (быстрые векторные операции NumPy)
y_coords = vertices_array[1::3]
t = (y_coords - min_y) / (max_y - min_y)
gray = 0.3 + t * 0.6
colors_array[0::3] = gray
colors_array[1::3] = gray
colors_array[2::3] = gray
```

Сравнение времени загрузки и инициализации нашей программы с другими популярными 3D просмотрщиками.  
**Для нашей программы указано:** `время парсинга / общее время инициализации`

| Модель | Blender | 3ds Max | DasViewer | F3D | **Наша программа** |
|--------|---------|---------|-----------|-----|--------------------|
| cube.obj | <0.5 с | <1 с | <1 с | <0.5 с | **0.008 с / 0.513 с** |

# Текущие ограничения и будущие планы

### Текущие ограничения

| Ограничение | Описание |
|-------------|----------|
| Отсутствие модели освещения | Программа использует только цвета вершин и текстуры. Модели выглядят "плоско", нет эффектов диффузного или зеркального освещения (Phong shading). |
| Отсутствие поддержки вершинных нормалей | Парсер игнорирует поле `vn` в OBJ файлах. Это делает невозможным качественное освещение. |
| Медленная загрузка текстур | Текстуры загружаются синхронно через Pyglet. Для моделей с текстурами высокого разрешения (4096×4096) время инициализации достигает 24 секунд. |
| Оптимизация загрузки текстур | Текущий конвейер: PNG/JPG → CPU декодирование → преобразование в RGB/RGBA → преобразование в формат GPU → передача в GPU. Это неэффективно. |

### Будущие планы

| План | Описание |
|------|----------|
| **Модель освещения Phong** | Добавление диффузного (diffuse) и зеркального (specular) освещения для более реалистичного рендеринга. |
| **Поддержка вершинных нормалей** | Полная поддержка поля `vn` в OBJ парсере для корректного расчета освещения. |
| **Новые типы текстур** | Поддержка карт нормалей (normal maps), карт зеркального отражения (specular maps) и других типов текстур. |
| **Оптимизация загрузки текстур** | Добавление отдельного **препроцессора текстур**, который будет заранее конвертировать PNG/JPG в GPU-нативные форматы (DDS/KTX). При запуске программа будет напрямую загружать уже готовые DDS/KTX текстуры, минуя медленный этап CPU декодирования и преобразования. Ожидаемое ускорение — до **10 раз**. Это позволит принципиально устранить самое узкое место в производительности текущей версии. |
| **Поддержка анимации** | Реализация чтения последовательностей OBJ файлов или встроенных кадров анимации. |
